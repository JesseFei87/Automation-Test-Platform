# ICM AI 自动化测试平台状态审计

日期：2026-06-05

## 当前已经真实实现

- 前端 5 个页面已落地，并支持左侧导航切换。
- 后端已有 FastAPI、SQLite、后台 worker、AIService 边界。
- `/api/cases` 可读取 `test-cases/icm/*.yaml`，当前识别 12 条 case。
- `/api/reports` 与 `/api/reports/{run_id}` 可读取历史 Markdown 报告。
- 报告详情接口可返回最新报告、截图路径和 AIService 分析结果。
- 执行页已可提交 `run-case` / `run-batch` 任务，并轮询任务详情日志。
- 单 case runner 已支持平台传入固定 `run_id`，便于报告和任务追踪对齐。

## 仍偏静态或半实现

- Dashboard 仍主要展示 mock 汇总，没有聚合真实 case、run、report 指标。
- 需求工作台可调用分析接口，但测试点编辑、优先级修改、覆盖状态保存还没有做。
- 用例工具箱可读取 case 和生成 YAML 草稿，但“确认入库正式 YAML”还没有做。
- 执行中心已经接队列和日志接口，但还没有做取消任务、失败重试、批跑进度拆分。
- 报告详情能读取最新报告，但还没有报告列表筛选、点击切换 run、截图在线预览。
- AIService 当前是可替换边界，Minimax M3 真实 HTTP 调用尚未接入。
- 截图文件已有后端接口，但前端还主要显示阶段占位缩略图，没有渲染真实图片。

## 下一步建议

优先做“报告与执行证据闭环”，而不是继续扩页面：

1. Dashboard 改为真实指标聚合。
2. 报告中心增加报告列表、run 选择、真实截图预览。
3. 执行中心补批跑子 case 进度和失败重试入口。
4. 需求/用例链路再补保存与正式 YAML 落盘。
5. 最后接 Minimax M3 真实接口。
