# Lessons

- 执行任务时，过程文档统一放在 `doc/`。
- 分析网络入口时，优先检查 URL、证书、跳转链路和反向代理风险，不直接用临时 `IP:端口` 下结论。
- 发现逻辑矛盾或环境风险时，先记录证据，再继续扩展性操作。
- 用户纠正 case 目标后，先更新 `lessons.md` 和对应 case 资产，再恢复执行。
- `TC-ICM-006` 这类“新增后继续被后续 case 依赖”的创建型用例，不能只校验记录存在；创建时要把关键业务状态一并设置到预期值，本例里设备状态必须显式启用。
- 2026-06-05 测试点思维导图应接入左侧测试点菜单，不放在需求工作台内；需求工作台继续负责需求输入和 AI 分析。
