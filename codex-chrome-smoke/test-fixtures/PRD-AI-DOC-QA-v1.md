# PRD-AI-DOC-QA：AI 文档问答（单文档） v1.0

> 用途：ICM 平台 E2E 测试用样例需求文档
> 创建人：许清楚
> 创建日期：2026-06-10
> 状态：Review
> 范围：聚焦"单文档问答"子集，验证 LLM 流式输出 + 引用溯源
> 不在范围：多文档跨库、团队标注、模型微调、API 开放

---

## 1. 产品目标
为中大型互联网公司的产品/运营/客服提供"上传 PDF/Word → 自动切片建索引 → 自然语言提问 → 获得带原文引用的流式答案"工作台，解决长篇文档找信息慢、问答模型无据可依两大痛点。MVP 仅支持单文档模式（一次会话绑定一个文档），目标是把"上传到首条答案"的端到端链路做扎实，作为后续多文档 RAG 的底座。

## 2. 用户故事
- **US-01** 作为**新注册用户**，我希望上传 PDF 后能在 60 秒内进入可提问状态，以便快速验证产品是否值得用。
- **US-02** 作为**研究员**，我希望向已上传文档提问时答案能逐字流式输出，并在每段后附原文页码与高亮片段，以便判断答案是否可信。
- **US-03** 作为**付费用户**，我希望单次问答可处理最长 8K token 的问题且不限制会话轮次，以便做长上下文研究。
- **US-04** 作为**企业管理员**，我希望能看到所有成员的文档用量、问答次数与剩余配额，以便做成本管控。
- **US-05（负面）** 作为**普通用户**，当我上传 0 字节空文件或可疑 PDF 时，系统应**明确拒绝并给出可执行的修正指引**，而非进入"永远解析中"。

## 3. 功能需求池

### P0 - 必须有
- **F-P0-001** 单文档上传：支持 `.pdf/.docx/.md/.txt`，单文件 ≤ 50 MB、≤ 500 页；超限必须客户端立即拒绝并提示具体超限项。
- **F-P0-002** 异步解析：上传成功后返回 `document_id`，后台执行；前端轮询 `/api/documents/{id}/status`（每 2 秒）展示 `queued → parsing → embedding → ready/failed` 五态。
- **F-P0-003** 自动切片：默认 500 token 一片、相邻 50 token 重叠；解析完成返回 `chunk_count` 并在 UI 显示。
- **F-P0-004** 单文档问答：选定 `ready` 文档后输入问题，前端用 SSE（`text/event-stream`）订阅 `/api/qa/stream`，逐字渲染答案。
- **F-P0-005** 引用溯源：每段答案末尾必须以 `[doc:doc-001#chunk-12 p.7]` 形式标注来源；点击引用可滚动到右侧原文片段并高亮。
- **F-P0-006** 历史会话：每个文档下保留最近 50 轮问答，支持按时间倒序加载与单条删除；会话标题默认取首问前 30 字，用户可手动重命名。
- **F-P0-007** 失败诊断：解析失败/切片失败/检索为空/模型超时四类错误必须返回结构化 `error_code`（如 `DOC_PARSE_FAIL`、`EMPTY_RETRIEVAL`），前端给出对应文案与"重试/换文档"按钮，且在文档管理页对 `failed` 状态行高亮红色警示。

### P1 - 应该有
- **F-P1-001** 引用复制：一键复制整段答案为 Markdown，自动包含引用角标。
- **F-P1-002** 模型降级：默认 `gpt-4o`，主模型连续 2 次超时（>10s）自动切到 `gpt-4o-mini` 并在答案头部加 `（已降级）`。
- **F-P1-003** 敏感词拦截：答案生成后正则扫描，命中替换为 `***` 并在气泡下显示"已过滤 N 处"；词库每月由内容安全团队统一更新。
- **F-P1-004** 文档删除：用户可删除自己上传的文档，同步清除向量索引与会话历史，不可恢复；删除前需弹窗二次确认且显示该文档关联的会话数量。

### P2 - 可以有
- **F-P2-001** 主题模式：问答页支持浅色/深色切换。
- **F-P2-002** 导出 PDF：单次问答可导出为 PDF 报告（含引用角标与原文片段截图）。

## 4. UI 设计描述（文字版 wireframe）

**4.1 首页（`/#/home`）** 顶 64px 导航（Logo/工作台/文档库/用量/头像），左 200px 侧边栏（最近文档 + "上传新文档"主按钮），右主区域为欢迎卡片 + 3 个空状态引导（首篇文档/示例文档/使用指南）。

**4.2 上传页（`/#/upload`）** 上方 480×240 拖拽区（虚线框 + "支持 PDF/DOCX/MD/TXT"），下方上传队列表（文件名/大小/进度/状态/取消），右下"开始上传"按钮在队列非空时高亮。

**4.3 问答页（`/#/qa/:document_id`）** 双栏布局。左 60% 对话流（用户问题右对齐气泡、AI 答案左对齐且逐字渲染，气泡底显示"复制/重新生成/反馈"，每段下方有可点击引用角标）；右 40% 原文对照面板（默认显示当前引用所在页，命中片段黄色高亮，可翻页）。底 80px 固定输入区（多行 textarea + 发送 + "当前文档:xxx.pdf"标签）。

**4.4 文档管理页（`/#/documents`）** 表格视图，列：文件名/大小/状态/切片数/上传时间/操作（打开/删除），顶部按状态筛选与按时间排序，批量删除勾选框。

**4.5 历史记录页（`/#/qa/:document_id/history`）** 按时间倒序的会话卡片流，每卡片显示首问摘要、问答轮次、最后活跃时间，点击展开进入对话页（保留上下文）。

## 5. 验收标准
- **AC-01** Given 用户上传 5MB PDF，When 上传完成且解析结束，Then 状态 `ready` 且 `chunk_count>0`，全程 ≤ 60 秒。
- **AC-02** Given 文档 `ready`，When 用户输入"第三章的核心结论是什么"并发送，Then 1.5 秒内出现首 token，答案中至少 1 条引用角标。
- **AC-03** Given AI 返回带引用答案，When 点击 `[doc:doc-001#chunk-12 p.7]`，Then 右侧原文面板在 500ms 内滚到第 7 页并高亮。
- **AC-04** Given 用户上传 0 字节文件，When 点击上传，Then 1 秒内弹 Toast「文件为空，请重新选择」且不发网络请求。
- **AC-05** Given 用户上传 60MB PDF（>50MB 限制），When 选文件，Then 客户端立即阻断并提示「文件超过 50MB 限制（当前 60MB）」。
- **AC-06** Given 主模型连续 2 次超时，When 触发第 3 次问答，Then 自动切 `gpt-4o-mini` 且答案开头含 `（已降级）`。
- **AC-07** Given 用户删除文档，When 确认，Then 状态变 `deleted`、向量索引清除、会话清空，二次访问 URL 返回 404。
- **AC-08** Given 50 用户同时对同一文档问答，When 负载上升，Then P95 首 token 时延仍 ≤ 1.5 秒、错误率 ≤ 1%。

## 6. 边界与异常
- **E-01** 空文件（0 字节）：前端阻断，错误码 `FILE_EMPTY`。
- **E-02** 超大文件（>50MB 或 >500 页）：前端阻断，错误码 `FILE_TOO_LARGE` / `PAGE_OVERFLOW`。
- **E-03** 乱码/加密 PDF：解析阶段失败，错误码 `DOC_PARSE_FAIL`，前端提示"文件可能已加密或损坏"。
- **E-04** 重复上传同名：允许但 `document_id` 唯一；UI 文件名后追加 `(2)` 避免混淆。
- **E-05** SSE 断流：15 秒未收 token 显示"连接中断，点击重试"，重试从断点续传。
- **E-06** 并发限制：单用户最多 3 个并发 SSE 连接，第 4 个被拒并提示"已有问答进行中"。
- **E-07** 检索为空：top-5 相似度均 < 0.3 时返回固定文案"未在文档中找到相关内容，建议换个问法"，不调生成模型。
- **E-08** 敏感词：答案生成后正则扫描，命中替换为 `***` 并在气泡下显示"已过滤 N 处"。

## 7. 业务规则与约束
- **B-01** 免费用户：每月 50 次问答、20 篇文档、每篇 ≤ 10MB。
- **B-02** 付费用户：每月 5000 次问答、500 篇文档、每篇 ≤ 50MB。
- **B-03** 引用角标格式固定 `[doc:{document_id}#chunk-{index} p.{page}]`，缺失页码用 `p.-` 兜底。
- **B-04** 状态机：`queued → parsing → embedding → ready`，任一阶段失败入 `failed` 终态；`ready` 可转 `deleted`，其他状态不可逆。
- **B-05** 会话上下文：同文档连续问答保留最近 6 轮，超出仅保留当前问题。
- **B-06** 计费：按"问答发起次数"计费一次，模型降级、流式中断重连不计费；空问题或检索为空场景不消耗次数。
- **B-07** 数据保留：免费 30 天、付费 365 天，到期前 7 天邮件提醒；到期后文档状态自动转 `expired`，向量索引同步清除。

## 9. 性能与可用性指标
- **NFR-01** 首屏 LCP ≤ 2.0 秒（P75，4G 网络）。
- **NFR-02** 问答首 token ≤ 1.5 秒（P95），完整答案 ≤ 8 秒（P95）。
- **NFR-03** 解析吞吐：单文档解析+嵌入 ≤ 60 秒（95% 分位，50MB PDF）。
- **NFR-04** 并发：单集群 ≥ 50 QPS 流式问答，P99 错误率 ≤ 1%。
- **NFR-05** 可用性 SLA 99.9%（月度停机 ≤ 43 分钟），解析与问答链路独立计 SLA。

## 10. 待确认问题
- **Q-01** F-P0-001 的"500 页"上限是按 PDF 实际页数还是解析后切片映射页数？建议以实际页数为准。
- **Q-02** F-P1-002 模型降级是否需要前端可配置开关，还是仅服务端自动？当前倾向全自动。
- **Q-03** F-P0-007 失败诊断在 `failed` 状态下是否允许"重新解析"？还是只能删除重传？建议允许重试一次。
- **Q-04** 敏感词词库由谁维护、是否支持租户自定义？MVP 建议用平台统一词库。
- **Q-05** 引用角标 `p.7` 在 Word/MD（非分页 PDF）下如何映射？建议 MD/TXT 用 `line:120`，Word 暂不支持页码定位。
